Drawing overlays are `OverlayTemplate` instances. The built-in set is registered
automatically when `registerExtensions: true` (the default), so you can select
any of them through [`useDrawingTools`](../../hooks/use-drawing-tools/).

## Try it

Pick a tool, then click on the chart to draw. Press the same button again to
deactivate.

```ts
// overlays === [arrow, circle, rect, triangle, …] (26 items)
```

## Manual registration

If you disable the default extensions (`registerExtensions={false}`), register
them yourself — `registerExtensions()` is idempotent:

```ts
import { registerExtensions } from "react-klinecharts-ui";

registerExtensions(); // repeated calls are ignored
```

## Feature overlays

The feature overlays `orderLine`, `alertLine` and `depthOverlay` are **registered
automatically** by `registerExtensions()` (default), so `useOrderLines` and
`useAlerts` work out of the box with no provider wiring. Passing them via the
provider's `overlays` prop still works and is harmless:

```tsx
import { orderLine } from "react-klinecharts-ui";
import { depthOverlay } from "react-klinecharts-ui/extensions";

// Optional — these are already auto-registered.
<KlinechartsUIProvider overlays={[orderLine, depthOverlay]}>…</KlinechartsUIProvider>;
```

`orderLine` is the building block for [`useOrderLines`](../../hooks/use-order-lines/);
`alertLine` is drawn by [`useAlerts`](../../hooks/use-alerts/); `depthOverlay`
renders order-book liquidity bars at each price level (update via
`chart.overrideOverlay({ id, extendData })`). See the
[Order lines guide](../../guides/order-lines/) for a worked example.
